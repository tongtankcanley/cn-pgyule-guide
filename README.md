# cn-pgyule-guide

本仓库用于归档和发布多个独立 HTML 页面，内容涵盖通用信息整理与展示。仓库本身不针对任何特定域名或具体网站，仅为页面文件的结构化存放与版本管理提供支持。

## 项目简介

`cn-pgyule-guide` 旨在以静态 HTML 页面的形式，汇集并长期维护一批独立的信息页面。这些页面可能涉及知识整理、工具导航、参考文档等不同方向，所有页面均以纯前端形式构建，无需后端服务即可直接浏览。

仓库的维护原则是：保持页面内容的中立、准确与可访问性，不包含任何形式的商业推广、诱导点击或具体网站推荐。

## 目录说明

```
cn-pgyule-guide/
├── index.html          # 仓库首页入口，提供页面导航
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-example1.html
│   ├── page-example2.html
│   └── ...
├── assets/             # 共用资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的默认入口，可快速跳转到各独立页面。
- `pages/`：所有归档的 HTML 页面均放置在此目录下，每个页面应保持独立、自包含。
- `assets/`：存放页面共用的静态资源，如样式表、脚本文件、图片素材等。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接通过浏览器打开，无需额外环境配置。
- 页面内容遵循通用信息展示规范，不包含具体域名、网址、营销承诺或诱导点击元素。
- 页面之间可能存在交叉链接，但链接仅指向本仓库内的其他页面或资源文件。
- 归档的页面会定期进行内容复核，以确保信息准确性与可读性。

## 维护说明

- 本仓库由维护者不定期更新，新增页面或修正已有内容时，会通过 commit 记录变更。
- 欢迎提交 Issue 或 Pull Request 来建议新增页面主题、报告内容错误或提出改进建议。
- 所有贡献需遵守本仓库的中立、非营销原则，不得插入任何商业推广或外链。
- 维护者保留对页面内容进行审核、合并或拒绝的权利。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/cn-pgyule-guide.git
   ```
2. 直接打开 `index.html` 即可浏览页面导航。
3. 也可直接打开 `pages/` 目录下的任意 HTML 文件查看具体内容。
